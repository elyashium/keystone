# Local AI Setup Guide

This guide explains how to set up and use the local AI features (Privacy Mode) in Keystone.

## Overview

Privacy Mode allows you to process documents and chat completely offline using local AI models. This ensures your data never leaves your device and provides full privacy.

## Features in Privacy Mode

- **Local PDF Processing**: Extract text and generate knowledge graphs locally
- **Local Embeddings**: Generate semantic embeddings using Transformers.js
- **Local LLM**: Answer questions and generate summaries using local language models
- **Local Vector Storage**: Store and search document chunks in a local SQLite database
- **Local Chat History**: Keep conversation history in a local database

## Prerequisites

1. **Install Dependencies**: Run `npm install` to install the new dependencies
2. **Download a Local LLM Model**: Download a GGUF format model file

## Setting Up a Local LLM

### Option 1: Lightweight Model (Recommended for most computers)
Download **Phi-3-mini** (small, fast, good quality):
- Go to: https://huggingface.co/microsoft/Phi-3-mini-4k-instruct-gguf
- Download: `Phi-3-mini-4k-instruct-q4.gguf` (about 2.4GB)

### Option 2: Higher Quality Model (For powerful computers with 16GB+ RAM)
Download **Llama 3 8B**:
- Go to: https://huggingface.co/Meta-Llama/Meta-Llama-3-8B-Instruct-GGUF
- Download: `Meta-Llama-3-8B-Instruct-Q4_K_M.gguf` (about 4.6GB)

### Model Installation
1. Start the app once to create the necessary directories
2. The app will show the models directory path (usually in `%APPDATA%/keystone-desktop/local-ai/models/` on Windows)
3. Copy your downloaded `.gguf` file to this directory
4. Restart the app
5. Toggle "Privacy Mode" on the home page

## Usage

### Enabling Privacy Mode
1. On the home page, click the "Cloud Mode" button in the top-right
2. It will switch to "Privacy Mode" and initialize local services
3. Wait for the status to show "Local AI Ready" (first time may take 1-2 minutes)

### First-Time Setup
- **Embedding Model**: Downloads automatically (about 90MB) on first use
- **LLM Model**: You must download manually (see above)
- **Initialization**: First run takes longer as models load into memory

### Using Local AI
Once Privacy Mode is active:
- Upload PDFs normally - they'll be processed locally
- Knowledge graphs generated by local LLM
- Chat and ask questions - answered by local LLM
- Generate summaries - created by local LLM
- All data stays on your device

## Troubleshooting

### "No GGUF model file found"
- Download a model file (see setup instructions above)
- Ensure the `.gguf` file is in the correct models directory
- Restart the app

### Slow Performance
- Use a smaller model (Phi-3-mini instead of Llama 3 8B)
- Close other applications to free up RAM
- Consider upgrading RAM if using larger models

### Out of Memory Errors
- Switch to a smaller quantized model
- Reduce context size in the model settings
- Restart the app to free up memory

### Backend Won't Start
- Check that all dependencies are installed (`npm install`)
- Look for error messages in the developer console (F12)
- Try restarting the app

## Model Recommendations by System

### 8GB RAM or less
- Use: `Phi-3-mini-4k-instruct-q4.gguf`
- Performance: Good for most tasks, fast responses

### 16GB RAM or more
- Use: `Meta-Llama-3-8B-Instruct-Q4_K_M.gguf`
- Performance: Higher quality responses, slower but still reasonable

### Advanced Users
- Try other models from Hugging Face with GGUF format
- Adjust quantization levels (Q4_K_M, Q5_K_M, Q8_0) based on quality vs speed needs

## Data Storage

Local data is stored in:
- **Windows**: `%APPDATA%/keystone-desktop/local-ai/`
- **macOS**: `~/Library/Application Support/keystone-desktop/local-ai/`
- **Linux**: `~/.config/keystone-desktop/local-ai/`

This includes:
- Vector embeddings database
- Chat history database
- Model cache files

## Privacy Benefits

When using Privacy Mode:
- ✅ No data sent to external servers
- ✅ No internet connection required (after initial setup)
- ✅ Full control over your data
- ✅ No usage tracking or logging
- ✅ Works completely offline

## Support

If you encounter issues:
1. Check the console for error messages (F12 in the app)
2. Verify model file is correctly placed
3. Ensure sufficient RAM for your chosen model
4. Try restarting the application

The local AI features require significant computational resources but provide complete privacy and offline functionality.
